# Phased roadmap

**Last updated:** 2026-07-07  

Two timelines: **industry-norm** (teams, regulation, adoption) vs **founder-velocity** (assuming continued speed + AI leverage + tight scope). Use founder column for *software you can demo*; use industry column for *mass replacement of iOS + banks + elections*.

## Phase 1 — Prove the platform (OS 0.1)

**Goal:** One daily-driver loop: **my identity, my server, my music, my devices.**

| Item | Founder-velocity | Industry-norm |
|------|------------------|---------------|
| Linux desktop live image + Sandbox shell | 2–4 months | 1–2 years |
| One mobile path (ROM or Linux phone niche) | +2–4 months | +1 year |
| Accounts = keys; server sync for follows/settings | 1–2 months | 6–12 months |
| Music station integrated in shell | reuse Music repo | same |
| Social v0: follows + E2E messages | 2–3 months | 1 year |
| Builder v1: offline signed station packages | 2–4 months | 1–2 years |

**Founder total (focused):** ~**6–18 months**  
**Don’t boil the ocean:** no banking, no vote, no marketplace yet — only what proves sovereignty daily.

---

## Phase 2 — Federation (OS 0.5)

**Goal:** **Your node** talks to **friend nodes** worldwide (with consent).

| Item | Founder-velocity | Industry-norm |
|------|------------------|---------------|
| Follows/likes on tier34 + cross-device sync | 1–2 months | 6 months |
| Server-to-server outbox protocol | 3–6 months | 2–3 years |
| Overlay default (self-hosted Headscale/Nebula docs → productized) | 2–4 months | 1 year |
| Marketplace station v0 (locker listings, LAN/trusted friends) | 3–6 months | 2 years |
| Banking vault v1 (one P2P rail, community scale) | 6–12 months | 3–5 years |

**Founder total:** ~**1–3 years** after Phase 1  
**Clock that doesn’t speed up:** user adoption, legal compliance for money.

---

## Phase 3 — OS in the public sense

**Goal:** Installer, updates, Builder ecosystem, governance modules.

| Item | Notes |
|------|--------|
| “Sandbox OS” installer / updates | Build: 1–2 years at founder pace |
| Marketplace at scale (federated discovery) | Needs Phase 2 protocol + moderation tools |
| Voting (communities → larger) | Crypto + process; national = society-scale |
| Mass “leave Windows/iPhone” | **Adoption clock** — unbounded |

**Industry 7+ years** still applies to **replacing default OS for hundreds of millions**, not to **having a credible sovereign stack for early adopters**.

---

## Wedge order (recommended)

1. **Music** — daily use (done)  
2. **Identity + server sync** — same person, all devices  
3. **Social primitive** — follow + message  
4. **Marketplace** — locker listings (freedom-aligned commerce)  
5. **Vault** — payments between known peers  
6. **Vote** — small groups first  
7. **Builder** — parallel track once station API stabilizes
